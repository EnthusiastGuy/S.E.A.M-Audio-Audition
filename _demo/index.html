ℹ S.E.A.M — Help & Reference

What this app does

S.E.A.M (Segmented Evaluation & Audition Module) is a browser-based player for dynamic / branching music stored as WAV segments. It builds a timeline from your filenames, plays transitions in order, supports per-part looping and crossfades between tracks, and can export stitched mixes or single files — all on your machine.

Privacy & requirements

  • Audio is read from folders you choose; nothing is uploaded to a server.
  • Use a Chromium-based browser (Chrome, Edge, etc.) with the File System Access API so you can pick folders and reopen recent projects.
  • MP3 / OGG export uses encoder scripts bundled with this app (no network). WAV export is always local once buffers are loaded.

Opening a project

  • Select Folder — choose the root of your sample pack: the folder that contains your song subfolders with .wav files. Songs will be directly inside the selected root.
  • Recent projects — up to ten previously opened folders appear on the welcome screen. Pick one to restore permission and reload (handles are stored in IndexedDB).
  • Change Folder — after loading, use the header dropdown to switch to another recent project, Browse… for a new folder, or stay on the current one.

Folder layout & discovery

Inside the selected root (or inside its optional wav/ subdirectory), each subfolder name is a song title. Inside each song folder, only .wav files are used. The app scans those files and groups them by the naming rules below.

Song list

Each row shows number, name, duration, part count, and actions (play / export when available).

  • Drag rows to reorder the playlist.
  • If the Song Name.wav main full mix is missing, the song still appears and the timeline defaults to all parts in order 1..n.
  • Songs with no main file and no parts are omitted entirely.
  • Click the row (anywhere except the action buttons) to open or close the part sheet for that song.

Playing & queue behaviour

  • ▶ Play starts playback; controls become ⏸ Pause and ■ Stop, and the seek bar appears under the row.
  • Several songs can play at once — useful for layering cues.
  • When a song ends, the next row in playlist order starts. After the last song, playback wraps to the top.
  • The Crossfade knob (sidebar) overlaps the tail of the current song with the start of the next: when less than N seconds remain, the next track fades in while the current one fades out.

Crossfade knob

Range 0.0 – 20.0 seconds in 100 ms steps. Drag the indicator along the top arc, use the mouse wheel over the knob, or click the tick marks at 0 / 5 / 10 / 15 / 20 s. The value label shows seconds with one decimal place.

Seek bar

  • Coloured bricks show each segment in proportion to its duration along the edited timeline.
  • Click a brick to jump to that segment’s start.
  • Drag the white handle to seek anywhere.
  • The time above the handle is cumulative played time, including loop repeats.
  • The percentage on the right is position versus the timeline length without extra loop passes (the “unlooped” layout length).

Loop controls

A Loop button appears under a brick only when that part’s metadata says it may transition to itself (its own part number appears in the “next” list after the arrow). Open the menu to choose:

  • Continue (once) — play through once, then follow the timeline.
  • 2× … 10× — repeat that segment N times, then continue.
  • Forever ∞ — repeat until you seek away.

The button text updates during playback (e.g. Loop 2/3).

Part sheet & timeline editor

With the part sheet open you see:

  • File list — every WAV for that song (full mix and segments), each with play once, play looped (play + loop icon), Seam preview (orange play + loop), stop, optional mini seek bar and waveform when a row is soloing, and per-file download format.
  • Seam preview — for auditioning loop seams: plays the beginning and end of the file (each for a duration you set in Settings), and fast-forwards the middle so you hear the join repeat. During the fast-forward hop, volume is lowered so the sped-up audio is less harsh. The mini bar highlights in orange for that hop; seeking on the mini bar is disabled while Seam preview is active.
  • Timeline — the same bricks as the main seek bar; drag bricks horizontally to reorder; drag from the list onto the brick row to insert a segment between others.
  • Hover a list entry to highlight matching bricks (and the reverse).

Export & downloads

  • Stitched composition — the download button beside transport (folder icon) renders the current timeline (including your reordering) to one file. It is only shown when a Song Name.wav main full mix exists for that title.
  • Choose format with the ▾ menu: WAV, MP3, or OGG. Your choice is remembered per song.
  • Per-part download — in the part sheet, download any single file (main or segment) in WAV / MP3 / OGG.
  • 60-minute limit — if the stitched timeline would exceed one hour, preview rendering is blocked and you’ll see a message; shorten the arrangement and try again.

Settings

Open the gear (⚙) in the header for MP3 (bitrate, sample rate, channels), OGG Vorbis (quality, sample rate, channels), timeline performance (max clip length for drawing waveforms), and Seam preview length (milliseconds played at the start and end of a part in Seam preview; default 2000 ms). Settings are saved per project (same folder name) with the rest of the session. Reset defaults restores built-in starting values.

Playback speed

The Playback speed knob applies to all active audio. Clockwise increases speed; counter-clockwise slows, stops, or reverses.

  • North-East: 100% (default)
  • South-East: 200% (max forward)
  • North: 0% (stopped)
  • North-West: −100% (reverse)
  • South-West: −200%
  • South: −250% (max reverse)

Values snap in 5% steps with a short click sound. You cannot jump straight from max forward to max reverse — turn through the north side of the dial.

Brick Playground (snap-and-build view)

Switch to Playground view with the sidebar VIEW toggle (brick icon). This is a freeform canvas where you can build new sequences by snapping parts from any song, audition those clusters, and export them.

  • Template rows (combs) — each song appears as a parking row of template bricks with a vertical comb spine. Drag the spine to move that whole song template row.
  • Create bricks — template bricks are source tiles: click to select; drag about 5px+ to spawn a duplicate into the workspace. Existing workspace bricks can also be duplicated with Ctrl/Cmd + drag.
  • Magnet snap — drag free bricks (or connected groups) near others to auto-snap edges into a cluster.
  • Ghost reorder — hold a free brick for about 1 second (without moving) to pick and move just that one brick out of its current sequence without dragging the whole cluster.
  • Seam insert marker — when dragging near the midpoint between two bricks, a marker appears; dropping there inserts the moved brick(s) between those neighbors in one step.
  • Delete selected cluster — with Playground focused, press Delete / Backspace to remove non-template bricks from the active cluster (template source bricks are protected).

Brick Playground cluster controls

Select a brick to show controls for its connected cluster:

  • Transport — Play once, Play looped, and Seam preview (same head/tail audition concept as part-sheet seam preview).
  • Seek bar — appears under the cluster while playing; drag to seek. During seam preview, seek is intentionally disabled.
  • Break apart — separates a connected cluster into spaced bricks on one row.
  • Remove duplicates (X icon) — keeps one of each duplicated sound inside the selected cluster.
  • Download cluster mix — renders the selected cluster left-to-right as one file. Use the format button to cycle WAV → MP3 → OGG.
  • Duration badges — connected clusters with 2+ free bricks show a total-duration badge above them.

Brick Playground navigation & shortcuts

  • Pan — drag empty canvas space.
  • Zoom — mouse wheel over the viewport (centered at pointer).
  • Undo / redo — Ctrl/Cmd + Z and Ctrl/Cmd + Y (also Ctrl/Cmd + Shift + Z for redo).
  • Reset positions — Playground Tools panel restores default comb layout plus default zoom/pan.

Session persistence

For each root folder, the app saves to localStorage: playlist order, crossfade, speed, per-song timeline edits, loop choices, export format preferences, encoding, waveform performance and seam preview length, which part sheets were left open, and Brick Playground state (view mode, brick positions, zoom/pan, and Playground download format). Reload the page and reopen the same folder to continue where you left off.

File naming convention

  • Full song (optional) — Song Name.wav (base name matches the folder; no part index).
  • Segment with branches — Song Name N -> A, B, C.wav or the same with a single hyphen before the arrow: Song Name N - A, B, C.wav. N is this part’s index; A, B, C are allowed next part indices.
  • Terminal segment — Song Name N.wav (number only, no arrow list) if there are no further branches.
